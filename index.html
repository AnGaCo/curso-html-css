Certificado Final del curso AP4.0

Parrafo 1

En esta certificación de Diseño Web Adaptativo, aprenderás los lenguajes que los desarrolladores usan para construir páginas web: HTML (Lenguaje de Marcado de Hipertexto) para el contenido, y CSS (hojas de estilo en cascada) para el diseño.

Parrafo 2

Primero, crearás una aplicación de fotos de gatos para aprender los conceptos básicos de HTML y CSS. Más adelante, aprenderás técnicas modernas como variables CSS, mediante la construcción de un pingüino, y las mejores prácticas para la accesibilidad mediante la construcción de un sitio de cuestionarios.

Parrafo 3

Por último, aprenderás cómo hacer páginas web que respondan a diferentes tamaños de pantalla, mediante la construcción de una galería de fotos con Flexbox, y un diseño de artículos de revista con CSS Grid.

Informacion

El elemento formulario HTML form representa una sección de un documento que contiene controles interactivos que permiten a un usuario enviar información a un servidor web.

Todos los formularios comienzan con un elemento form, como este: formacción="/my-handling-form-page" method="post"

El atributo action=” ” define la ubicación (URL) donde se envían los datos que el formulario ha recopilado cuando se validan…

El elemento form define dónde y cómo enviar los datos gracias a los atributos action=” ” y método.…

El tipo ( type=” ” ) es una característica de un valor que afecta al tipo de datos que puede almacenar; por ejemplo, en JavaScript un Boolean (en-US) solo tiene valores true/ false, mientras que un String contiene cadenas de texto, un Numbercontiene números de cualquier tipo, y así sucesivamente.

El elemento placeholder=” ” representa el texto provisional en un elemento input o un elemento textarea. Nota: En la mayoría de los navegadores, la apariencia del texto provisional es traslúcido o un color gris claro por defecto.

Required: Obligatoriamente los input deben tener el atributo required aplicado a ellos. Esto asegurará que las personas que naveguen con la ayuda de tecnología de asistencia, como un lector de pantalla, puedan comprender qué entradas necesitan contenido válido para garantizar un envío exitoso.

El elemento HTML input type="button" es una versión específica del elemento que se usa para crear un botón en el que se puede hacer clic sin ningún valor por defecto, es decir, no tiene un comportamiento predeterminado como por ejemplo. En HTML5 ha sido reemplazado por el elemento button, input, input type="reset" Los navegadores generan un controlador para un botón clicable sin valor por defecto. El botón puede contener cualquier texto. El controlador puede variar de un navegador a otro. El elemento HTML input se usa para crear controles interactivos para formularios basados en la web con el fin de recibir datos del usuario. Hay disponible una amplia variedad de tipos de datos de entrada y widgets de control, que dependen del dispositivo y el agente de usuario ( agente de usuario (en-US)). El elemento input es uno de los más potentes y complejos en todo HTML debido a la gran cantidad de combinación de tipos y atributos de entrada
